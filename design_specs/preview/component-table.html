FVT · v1.0 · Tabular data system

A primitive for ranked, scored, sequenced data.

Extracted from the league table and tournament bracket, fvt is one grid-based row primitive that handles standings, fixtures, brackets, leaderboards, head-to-heads, and stats grids — all responsive by container size, not viewport.

1. Anatomy

Every FVT table is a stack of grid rows sharing a single column template declared as --fvt-cols. Rows are display: grid, never inline-flow — so spacing, alignment, and direct-manipulation edits stay clean.

Index

Numeric position with optional trend arrow. Mono, tabular nums.

Identity

Glyph + name + secondary code. Truncates with ellipsis. Code hides < 480px.

Numeric

Right-aligned, mono, tabular. Sign-aware colour via data-sign.

Series

Inline pip sequence (form, recent results, sparkline). Truncates to 3 on narrow.

Status

Pill badge with state tokens: live, success, warn, upcoming.

Heat

Cell tinted by intensity (0–100). Use for stats grids.

2. The grid contract

The component sets the column template; the primitive doesn't care. Tracks that should hide on narrow widths collapse to 0px rather than display: none, so cell-to-track alignment stays correct. 

/* Component declares its template via custom property */
.lt-row, .lt-head { --fvt-cols: 28px 1fr 32px 0px 0px 44px 22px; }

/* Reveal hidden tracks at container width >= 480px */
@container fvt (min-width: 480px) {
  .lt-row, .lt-head { --fvt-cols: 36px 1fr 52px 48px 48px 84px 40px; }
}

Container queries — not viewport queries. A table inside a 320px sidebar stays compact even on a 4K display.

3. Responsive behaviour

Drag the resize handle at the bottom-right of the demo to see the table reflow. Three things happen as it shrinks: the secondary code line under each name disappears, the form sequence drops to its last 3 entries, and at the smallest sizes optional tracks (Goals For, Goals Against) collapse entirely.

4. Variant — match card

A match card is a 2-row FVT table with a meta strip below. Same row primitive, different column template (no index, no series — just identity + score). Result state (data-result) drives the win / loss / TBD treatment.

5. Variant — stat grid

For dense stats (head-to-heads, season heatmaps), the same row + identity cell pair with heat cells whose intensity is set via --fvt-heat. The grid template just changes track count.

6. State system

State lives on data attributes, never on classes — easier to change at runtime, impossible to "forget" to remove. Five primitives cover everything observed across the league + bracket: 

/* Row-level */
<a class="fvt-row" data-zone="champions">       // Left-edge accent rail
<a class="fvt-row" data-highlight>              // Favourite / featured
<a class="fvt-row" data-result="win|loss|tbd">   // Match outcome

/* Cell-level */
<span class="fvt-cell-num" data-sign="pos|neg|zero">
<span class="fvt-pip" data-state="pos|warn|neg|neutral">
<span class="fvt-badge" data-state="live|success|warn|upcoming">

7. Density

Three density modes — compact, default, comfortable — swap row height and horizontal padding via custom properties. Same column template, same row code. Stack the table in a sidebar widget or as a full desktop dashboard.
